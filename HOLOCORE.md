# HoloCore knowledge workflow

Run `holocore status` before broad orientation. For project work, use a fresh Atlas first to identify relevant files and relationships, search corresponding Archive notes second, query Animus third only when prior work or chats matter, and open exact source files last. Build one route plan and execute it once; never route a HoloCore search back into itself. Each subsystem may run at most once per request. Writes must be explicit and scoped. Promote only verified durable knowledge into Archive, update existing notes before creating duplicates, and record uncertain claims as questions rather than facts.

Archive = verified knowledge. Atlas = structural map. Animus = remembered history. World = project. Sector = area inside a project. Memory Shard = raw remembered fragment. Archive Entry = polished durable note. Signal = one mapped thing. Constellation = group of related mapped things.
